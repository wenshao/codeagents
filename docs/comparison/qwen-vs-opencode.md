### OpenCode Plan/Build 模式详解

**Plan 模式（架构师视角）：**
- 不直接修改代码
- 生成自然语言形式的实施计划
- 读取代码库，分析依赖关系
- 输出高层级的设计文档
- **只读模式**，安全无风险

**Build 模式（工程师视角）：**
- 执行 Plan 生成的计划
- 直接修改代码文件
- 运行测试和构建
- 需要**权限审批**
- **写入模式**，实际改动

## 总结

### 相似之处

1. **开源免费**：都是开源项目，核心功能免费
2. **MCP 支持**：都完整支持 MCP 协议
3. **LSP 集成**：都支持语言服务器协议
4. **IDE 支持**：都支持 VS Code、JetBrains 等主流 IDE
5. **审批机制**：都有完善的权限审批系统

### 核心差异

| 方面 | Qwen Code 优势 | OpenCode 优势 |
|------|---------------|---------------|
| **模型** | Qwen3-Coder 优化 | 75+ 模型选择 |
| **成本** | 每天2000次免费 | 完全免费+自备 API |
| **架构** | 单一智能体 | Plan/Build 双代理 |
| **语言** | 中文优化 | 国际化 |
| **集成** | 阿里云深度集成 | 多云平台 |
| **LSP** | MCP 扩展 | 20+ 内置语言 |

### 最佳选择建议

- **国内个人开发者**：Qwen Code（开箱即用）
- **需要模型灵活性**：OpenCode（75+ 选择）
- **团队协作/架构审查**：OpenCode（Plan 模式安全）
- **预算极度敏感**：OpenCode + DeepSeek（¥1/百万tokens）
- **阿里云生态用户**：Qwen Code（无缝集成）
- **Go 语言项目**：OpenCode（原生 Go 编写）
