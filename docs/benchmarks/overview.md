# 基准测试概览

本文档涵盖 AI 编程代理的基准测试，包括 SWE-bench 和其他评估指标。

## SWE-bench

### 什么是 SWE-bench？

SWE-bench 是一个用于评估 AI 代理在真实软件工程任务上表现的基准。它使用来自开源仓库的真实 GitHub issues。

- **网站**：[swebench.com](https://www.swebench.com/)
- **仓库**：[github.com/princeton-nlp/SWE-bench](https://github.com/princeton-nlp/SWE-bench)

## 按基准选择

### 基准性能领先

**SWE-bench 领先者**：SWE-agent (增强版)
- 最适合：研究、自主 bug 修复
- 权衡：较慢、设置复杂

### 真实世界性能

**Claude Code**：推理和可用性最佳平衡
- 最适合：复杂重构、架构
- 权衡：更高成本 (Opus)

### 成本效率

**Haiku/Mini 模型**：快速且便宜
- 最适合：探索、简单任务
- 权衡：较低质量

### 速度

**Codex CLI**：最快的终端代理
- 最适合：快速命令生成
- 权衡：功能有限

### 国内开发者

**Qwen Code**：免费额度高
- 最适合：中文开发者、日常使用
- 权衡：生态较小

**Kimi CLI**：双模式交互
- 最适合：Kimi 用户、shell 集成
- 权衡：预览版本

## 运行基准测试

### SWE-bench

```bash
# 安装 SWE-bench
pip install swebench

# 运行评估
swebench.evaluate \
  --model_name claude-opus-4 \
  --dataset_name princeton-nlp/SWE-bench_Verified
```

### 自定义评估

```bash
# 使用 mini-swe-agent 进行简单基准测试
python mini_swe_agent.py \
  --dataset your-benchmark.json \
  --model claude-haiku-4
```

## 结果解读

### 数字含义

- **Pass@1**：首次尝试成功率
- **解决率**：完全解决的问题百分比
- **部分得分**：一些基准给予部分得分

### 局限性

- 基准测试不能捕获所有真实场景
- 一些代理可能为基准过度优化
- 任务选择偏差影响分数
- 不同工具在不同任务上表现出色
