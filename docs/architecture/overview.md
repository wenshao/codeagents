# Code Agent 架构

本文档解释 AI 编程代理的工作原理。

## 核心组件

### 1. 代理循环

大多数编码代理遵循类似的模式：

```
┌─────────────────────────────────────────────────────────────┐
│                     代理循环                                  │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│   ┌──────────┐    ┌──────────┐    ┌──────────┐             │
│   │  输入    │ -> │  推理    │ -> │  行动    │             │
│   │(prompt)  │    │  (LLM)   │    │(工具调用)│             │
│   └──────────┘    └──────────┘    └──────────┘             │
│                                              │              │
│                                   ┌──────────▼──────────┐   │
│                                   │   观察结果          │   │
│                                   └──────────┬──────────┘   │
│                                              │              │
│                                   ┌──────────▼──────────┐   │
│                                   │   更新上下文        │   │
│                                   └──────────┬──────────┘   │
│                                              │              │
│                                              └──────┐       │
│                                                     │       │
│                                              ┌──────▼───────┐│
│                                              │  重复/完成   ││
│                                              └─────────────┘│
└─────────────────────────────────────────────────────────────┘
```

### 2. 常见模式

#### ReAct（推理 + 行动）

使用者：Gemini CLI、mini-swe-agent

```
1. 思考："我需要先读取文件"
2. 行动：read_file("main.py")
3. 观察：（文件内容）
4. 思考："我看到第 42 行有 bug"
5. 行动：edit_file("main.py", ...)
6. 观察：（成功）
```

#### 计划-执行-反思

使用者：Claude Code（计划模式）、Cline

```
1. 计划：创建详细计划
2. 确认：获取用户批准
3. 执行：执行计划步骤
4. 反思：审查结果
5. 调整：修复任何问题
```

#### 工具调用

使用者：大多数现代代理

```
{
  "type": "tool_use",
  "tool": "bash",
  "input": {"command": "pytest tests/"}
}
```

### 3. 关键架构决策

#### 单代理 vs 多代理

| 类型 | 优点 | 缺点 | 示例 |
|------|------|------|----------|
| 单代理 | 简单、可预测 | 并行有限 | Claude Code、Aider |
| 多代理 | 并行工作、专门 | 协调复杂 | Goose、Warp |

#### 单体 vs 复合

| 类型 | 描述 | 示例 |
|------|-------------|----------|
| 单体 | 单个 LLM 做所有事情 | Aider、mini-swe-agent |
| 复合 | 多个专门组件 | OpenHands、SWE-agent |

#### 同步 vs 异步

| 类型 | 优点 | 缺点 | 示例 |
|------|------|------|----------|
| 同步 | 可预测、易于调试 | 较慢 | 大多数 CLI 工具 |
| 异步 | 更快、并行 | 更复杂 | Warp、Claude Code（子代理）|

## 实现示例

### mini-swe-agent（100 行）

```python
class SimpleAgent:
    def __init__(self, model):
        self.model = model
        self.history = []

    def run(self, task):
        while True:
            # 从 LLM 获取下一个行动
            response = self.model.predict(
                task,
                self.history
            )

            # 执行行动
            result = self.execute(response.action)

            # 更新历史
            self.history.append((
                response.action,
                result
            ))

            # 检查是否完成
            if response.done:
                break
```

这是更复杂代理构建的基础。
